==================================================
PROJECT IDENTITY (NON-NEGOTIABLE)
==================================================

FPBible is a reader-facing Bible project that presents Scripture with clarity and authority.

Core commitments:
- World English Bible (WEB) text only (verbatim, unedited)
- Fulfilled covenant transition (AD 30–70)
- New Creation reality as present and complete
- Zero futurism
- Hebrew idiom and first-century audience relevance as default

FPBible is NOT a debate platform.

==================================================
ABSOLUTE RULE: READER-FACING CHAPTER PAGES
==================================================

READER-FACING BIBLE CHAPTER PAGES MUST NEVER:

- Explain interpretation method
- Mention “lens”, “framework”, “approach”, or “perspective”
- Use meta language (“this project reads…”, “we interpret…”, “from a fulfilled view…”)
- Include Preface-style guidance (“how to read this”, “interpretive approach”)
- Contain labels like “fulfilled clarification”, “lens note”, “zero futurism”
- Sound apologetic, defensive, instructional, or explanatory about theology

❌ NEVER explain HOW the text is read  
✅ ALWAYS state WHAT the text means  

Bible chapters must ASSERT meaning, not explain method.

Method explanation is allowed ONLY in:
- About pages
- Prophecy Hub pages
- Appendices / Essays
- Internal standards documents

==================================================
FAIL CONDITIONS (HARD STOP)
==================================================

A chapter FAILS immediately if it:
- Explains its interpretive approach
- Sounds like commentary about interpretation rather than interpretation itself
- Frames fulfillment as a “view” instead of fact
- Leaves futurist ambiguity
- Uses hedging language (“may”, “might”, “suggests”, “possibly”)

If any fail condition occurs, REWRITE the output.
