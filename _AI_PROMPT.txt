<div class="wrap">
  <h1>FPBible — Locked Chapter HTML Standard</h1>
  <h2>Tightened (Audit-Proof)</h2>

  <div class="note">
    <span class="pill">Non-negotiable</span>
    <span class="pill">WEB text only</span>
    <span class="pill">Zero futurism</span>
    <span class="pill">Strict section order</span>
    <p class="muted" style="margin:0.55em 0 0;">
      If a chapter page breaks any locked rule, it is a fail and must be corrected.
    </p>
  </div>

  <hr />

  <section id="brand">
    <h2>0) Branding Rule (Non-Negotiable)</h2>
    <div class="fail">
      <strong>Hard rule:</strong> Do not use any paraphrase-branding terms anywhere on chapter pages
      (title, headings, nav, body, footer, metadata).
      Use only FPBible / Fulfilled Bible wording if naming is needed.
    </div>
  </section>

  <hr />

  <section id="lens">
    <h2>1) Theological Lens (Site Audit Compliant)</h2>
    <ul>
      <li>All framing, notes, and commentary must be consistently Covenantal / Full Preterist.</li>
      <li><strong>Zero Futurism:</strong> if commonly treated as “still pending,” state fulfillment plainly:
        <ul>
          <li>NT: first-century covenant transition (AD 30–70), explicitly mention AD 70 where applicable.</li>
          <li>OT: fulfilled in Christ (Cross/New Covenant) and/or in the covenant transition climaxing in AD 70, without drift.</li>
        </ul>
      </li>
      <li><strong>Definitions (locked):</strong>
        <ul>
          <li>“The End” = end of the Mosaic age (AD 70).</li>
          <li>“Coming / Parousia” = Christ’s covenantal vindication/judgment on Jerusalem leading to transition (AD 70).</li>
          <li>“Kingdom” = present, settled New Creation reality now.</li>
        </ul>
      </li>
      <li>Banned: “already but not yet” phrasing; any “still to be fulfilled” implication.</li>
      <li>Audience relevance is mandatory: identify who is addressed and why time-markers are time-bound (“this generation,” “near,” “soon,” etc.).</li>
    </ul>
  </section>

  <hr />

  <section id="text">
    <h2>2) Text Integrity (WEB — Source Locked) — Primary Gate</h2>
    <div class="fail">
      <strong>Hard gate:</strong> If Scripture cannot be verified as World English Bible (WEB) from eBible,
      do not output guessed text.
    </div>
    <ul>
      <li>Scripture must be copied verbatim from:
        <code>https://ebible.org/web/BOOK##.htm</code>
        (example: <code>https://ebible.org/web/MAT03.htm</code>).
      </li>
      <li>No rewriting, modernizing, harmonizing, condensing, or punctuation “cleanup.”</li>
      <li>No mixing translations (WEB only).</li>
      <li><strong>Selective bolding (locked):</strong> In Scripture, use <code>&lt;strong&gt;</code> only for exact phrases listed in Annotations.
        Nothing else is bolded (exact match only).</li>
    </ul>
  </section>

  <hr />

  <section id="links">
    <h2>3) Link Rules (Locked)</h2>
    <ul>
      <li>Cross-references must link to WEB on eBible using: <code>https://ebible.org/web/BOOK##.htm</code></li>
      <li>Prophecy Hub links must be internal and formatted exactly:
        <code>&lt;a href="/Bible/Prophecy/index.html#anchor"&gt;Theme Name&lt;/a&gt;</code></li>
      <li>Do not invent external cross-reference sites.</li>
      <li>Do not invent Prophecy Hub anchors unless explicitly instructed.</li>
    </ul>
  </section>

  <hr />

  <section id="template">
    <h2>4) Format + Template (Locked)</h2>
    <ul>
      <li>Valid HTML5.</li>
      <li>Use the locked Revelation-1 style internal CSS/layout (cards/sections/nav/buttons). No external CSS or imports.</li>
      <li>Must include consistent top navigation and bottom navigation:
        Bible Index + Book Index + Previous + Next.
      </li>
    </ul>
  </section>

  <hr />

  <section id="files">
    <h2>5) File + Title Rules (Locked)</h2>
    <ul>
      <li><code>&lt;title&gt;</code> and <code>&lt;h1&gt;</code> must match exactly:
        <strong>{Book} {Chapter} — {Short Covenant-Themed Subtitle}</strong>
      </li>
      <li>Prev/Next internal links must match the repo naming scheme for that book folder.</li>
      <li>If unknown, follow the existing pattern for that book and remain consistent within the output.</li>
    </ul>
  </section>

  <hr />

  <section id="sections">
    <h2>6) Required Section Order (Locked)</h2>
    <p class="muted">Each section must be separated by <code>&lt;hr /&gt;</code>. No extra sections inserted.</p>

    <div class="grid">
      <div class="card">
        <h3>1) Immediate Context</h3>
        <p>One short paragraph: covenant-litigation / redemptive-historical scene-setting (no fluff).</p>
      </div>

      <div class="card">
        <h3>2) Scripture (WEB)</h3>
        <p>Full chapter inside <code>&lt;div class="scripture-text"&gt;</code>, verse numbers as <code>&lt;span class="v-num"&gt;</code>.
          Include eBible source URL under the heading (not inside Scripture text). Apply selective bolding only per Annotations.</p>
      </div>

      <div class="card">
        <h3>3) Fulfillment Notes (Strict)</h3>
        <p><strong>Exactly 3–5 lines</strong>. Each line is exactly one sentence. No quotes, no “maybe,” no mini-commentary.
          Sharpen timing language. OT notes must state fulfillment trajectory (Christ → Cross → New Covenant → AD 70).</p>
      </div>

      <div class="card">
        <h3>4) Annotations</h3>
        <p>Gray/soft box with key terms + brief definitions (1–2 lines each). The terms/phrases listed here are the only phrases that receive <code>&lt;strong&gt;</code> in Scripture.</p>
      </div>

      <div class="card">
        <h3>5) Fulfillment Commentary (Depth Required)</h3>
        <p><strong>Minimum 700 words</strong> (900+ preferred). Numbered subheadings. Connect imagery to covenant realities (Law/Temple/Priesthood/etc.).
          State first-century resolution plainly (AD 70 context where applicable). Calm tone, no sparring.</p>
      </div>

      <div class="card">
        <h3>6) Prophecy Hub Links</h3>
        <p>Bullet list of <strong>exactly 3</strong> internal themes (existing anchors only unless instructed).</p>
      </div>

      <div class="card">
        <h3>7) Cross-References</h3>
        <p>Bullet list of <strong>exactly 3</strong> WEB links (OT/NT mix as appropriate).</p>
      </div>

      <div class="card">
        <h3>8) New Creation Reality</h3>
        <p>Short, grounded “now” meaning: presence, access, identity, Spirit-life. No fear-based application.</p>
      </div>

      <div class="card">
        <h3>9) Reflection</h3>
        <p>One challenging devotional question. Not moralistic. Not fear-based.</p>
      </div>

      <div class="card">
        <h3>10) Footer (Locked)</h3>
        <p>Prev/Next + Bible Index + Book Index links, official Ko-fi button, and extended WEB attribution note.</p>
      </div>
    </div>
  </section>

  <hr />

  <section id="qc">
    <h2>7) Quality Control — Hard Fail Conditions (Pass/Fail)</h2>

    <div class="fail">
      <strong>Hard fail if any occur:</strong>
      <ul>
        <li>Scripture is not verifiably WEB from eBible.</li>
        <li>Any Scripture is edited or rewritten.</li>
        <li>Any futurist “still pending” implication remains.</li>
        <li>Fulfillment Notes are fewer than 3 lines or more than 5 lines.</li>
        <li>Any <code>&lt;strong&gt;</code> appears in Scripture that is not an exact Annotations phrase.</li>
        <li>Prophecy Hub links are not exactly 3, or Cross-References are not exactly 3.</li>
        <li>Missing required sections or wrong section order.</li>
        <li>Branding rule violated.</li>
      </ul>
    </div>

    <div class="note">
      <strong>Pass condition:</strong>
      <span class="muted">Text integrity is flawless, fulfillment is unmistakable, structure is exact, and navigation is consistent.</span>
    </div>
  </section>
